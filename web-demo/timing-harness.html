Timing Distribution (µs)

📖 Interpretation Guide

Tight, unimodal distribution: Good. The rejection loop iteration count is not the dominant timing factor. JS engine noise masks the signal.

Bimodal or multimodal distribution: Concerning. Each cluster likely corresponds to a different iteration count (2, 3, 4, ...). The spacing indicates per-iteration cost.

Long tail (right-skewed): Moderate concern. High-iteration signing operations are distinguishable but harder to exploit than bimodal.

Uniform/noisy: Good for security. The JS engine's execution environment adds enough noise to bury the signal.

⚠️ Timer Resolution Note

performance.now() resolution depends on Cross-Origin Isolation headers:

Cross-Origin-Opener-Policy: same-origin
Cross-Origin-Embedder-Policy: require-corp

Without these headers (e.g., on GitHub Pages), timer resolution is degraded to ~100µs. Run via npm run dev with Vite to get ~5µs resolution.
