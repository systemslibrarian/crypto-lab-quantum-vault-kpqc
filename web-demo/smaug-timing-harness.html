🔬 SMAUG-T Level 1 — Timing Side-Channel Harness

This harness tests for timing differences that could enable a KyberSlash-style attack against the FO transform's implicit rejection mechanism.

TEST A — CRITICAL

Valid vs Invalid Ciphertext Timing (KyberSlash Test)

Attack model: An attacker modifies the KEM ciphertext in localStorage and measures decapsulation time. If valid and invalid ciphertexts have distinguishable timing, the FO transform's implicit rejection leaks information → potential key recovery.

Valid Ciphertext Decapsulation

Invalid (Tampered) Ciphertext Decapsulation

📖 Test A Interpretation

Overlapping distributions + non-significant Welch t-test: Good. The FO implicit rejection is constant-time at measurable resolution.

Separable distributions + significant Welch t-test (p < 0.001): CRITICAL. The decapsulation leaks ciphertext validity — this is the KyberSlash attack surface. An attacker can distinguish valid from invalid ciphertexts and potentially recover the secret key via chosen-ciphertext queries.

TEST B

Keygen Timing Consistency

Purpose: Check for timing variance in secret key generation (sparse ternary / HWT sampling).

📖 Test B Interpretation

Tight unimodal: Good. HWT sampling is constant-time.

Bimodal or heavy-tailed: Concerning. May indicate rejection sampling variance in sparse secret generation.

TEST C

Encapsulation Timing Consistency

Purpose: Check for timing variance in key encapsulation (r vector sampling, CBD).

📖 Test C Interpretation

Tight unimodal: Good. CBD sampling and encryption are constant-time.

Multimodal: Concerning. May indicate data-dependent branching in polynomial operations.

⚠️ Timer Resolution Note

performance.now() resolution depends on Cross-Origin Isolation headers:

Cross-Origin-Opener-Policy: same-origin
Cross-Origin-Embedder-Policy: require-corp

Without these headers (e.g., on GitHub Pages), timer resolution is degraded to ~100µs. Run via npm run dev with Vite to get ~5µs resolution.
